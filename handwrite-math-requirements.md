# iPad手書き数式ノートWebアプリ 要件定義書

## 1. 結論

本アプリは、iPad Safariで使う無料前提のReact製Webアプリとして開発する。  
初期版では、手書き数式を画像として保存し、LaTeX入力をKaTeXで表示する。  
OpenAI APIなどのAI連携は将来拡張として後回しにする。

## 2. 前提

| 項目 | 内容 |
|---|---|
| アプリ種別 | iPad向けWebアプリ |
| フロントエンド | React |
| 数式表示 | KaTeX |
| 保存方式 | IndexedDB |
| 画面向き | 横向き優先 |
| デザイン | ノートアプリ風 |
| 初期版の課金 | なし |
| AI連携 | 将来対応。初期版では実装しない |
| 主な利用端末 | iPad |
| 主なブラウザ | Safari |

## 3. 目的

iPad上で数式を手書きし、学習記録として保存・確認できるWebアプリを作る。

初期版では、数式の自動認識や自動解答は行わない。  
まずは無料で動く手書き数式ノートとして、以下を実現する。

- iPadで数式を手書きできる
- 手書き内容を画像として保存できる
- LaTeXまたは通常テキストで数式を入力できる
- KaTeXで数式をきれいに表示できる
- 答えや解説を手入力で保存できる
- 保存した記録を一覧・詳細画面で確認できる
- 将来的にAI連携を追加しやすい構成にする

## 4. 対象利用者

| 利用者 | 内容 |

| 学習者 | iPadで数式や途中式を書いて記録する |
| 教師・保護者 | 手書き内容や答えを確認する |
| 開発者 | 将来、数式認識やAI解答機能を追加する |

## 5. 想定利用シーン

| シーン | 内容 |
|---|---|
| 数学学習 | 問題式、途中式、答えを記録する |
| 家庭学習 | iPadをノート代わりに使う |

## 6. 開発方針

| 方針 | 内容 |
|---|---|
| 無料優先 | 初期版では有料APIを使わない |
| React採用 | UIを部品化し、後から機能追加しやすくする |
| KaTeX採用 | 入力したLaTeXを軽量にプレビュー表示する |
| IndexedDB採用 | 画像データをブラウザ内に保存する |
| 横向き優先 | iPadを横向きにしてノートのように使う |
| ノートアプリ風 | 紙のノートに近い落ち着いた画面にする |
| AI連携は後回し | OpenAI APIなどは初期版に含めない |

| 数式入力 | LaTeXまたは通常テキストを入力できる | 必須 |

| 機能 | 理由 |

| Mathpix連携 | 有料寄りのため |
| MyScript連携 | 有料寄りのため |
| 自動数式認識 | 初期版では実装負荷が高いため |
| ログイン | 個人利用のプロトタイプでは不要なため |
| クラウド同期 | 初期版では端末内保存を優先するため |

## 8. 画面要件

### 8.1 ホーム画面

保存済みの数式ノートを一覧表示する。

| 項目 | 内容 |
|---|---|
| 新規作成ボタン | 新しい手書き数式ノートを作成する |
| 履歴一覧 | 保存済みノートを表示する |
| 検索欄 | タイトルやメモで検索する |
| 削除操作 | 不要なノートを削除する |

画面イメージ:

```text
+------------------------------------------------+
| 手書き数式ノート                                |
| [新規作成]  [検索: 二次方程式                 ] |
|                                                |
| 2026/05/26  二次方程式                          |
| 2026/05/26  微分の練習                          |
| 2026/05/25  分数計算                            |
+------------------------------------------------+
```

### 8.2 手書き入力画面

iPadで数式を書くメイン画面。

| 項目 | 内容 |
|---|---|
| タイトル入力 | 問題名やメモ名を入力する |
| ツールバー | 戻る、クリア、保存、ペン太さを配置する |
| キャンバス | 手書き入力を行う |
| 数式入力欄 | LaTeXまたは通常テキストを入力する |
| プレビュー | KaTeXで数式を表示する |
| 答え入力欄 | 答えや解説を入力する |

画面イメージ:

```text
+------------------------------------------------+
| タイトル: [二次方程式                         ] |
| [戻る] [クリア] 太さ: [---o---] [保存]          |
|                                                |
| +---------------------------+ +--------------+ |
| |                           | | LaTeX入力     | |
| |   x^2 - 5x + 6 = 0        | | x^2-5x+6=0   | |
| |                           | |              | |
| |                           | | プレビュー   | |
| |                           | | x² - 5x + 6  | |
| +---------------------------+ +--------------+ |
| 答え・解説: x = 2, 3                            |
+------------------------------------------------+
```

### 8.3 詳細画面

保存済みノートの内容を確認する画面。

| 項目 | 内容 |
|---|---|
| タイトル | 保存したタイトルを表示 |
| 手書き画像 | 保存したPNG画像を表示 |
| 数式プレビュー | KaTeXで数式を表示 |
| 答え・解説 | 保存した答えや解説を表示 |
| 編集ボタン | 編集画面に移動 |
| 削除ボタン | 確認後に削除 |

## 10. データ項目

### 10.1 数式ノートデータ

| 項目名 | 型 | 必須 | 内容 |
|---|---|---:|---|
| id | 文字列 | 必須 | ノートID |
| title | 文字列 | 必須 | タイトル |
| imageDataUrl | 文字列 | 必須 | 手書き画像のData URL |
| formulaText | 文字列 | 任意 | LaTeXまたは通常テキスト |
| answerText | 文字列 | 任意 | 答え・解説 |
| createdAt | 文字列 | 必須 | 作成日時 |
| updatedAt | 文字列 | 必須 | 更新日時 |

### 10.2 将来拡張用データ

初期版では使用しないが、将来のAI連携を想定して追加しやすい構造にする。

| 項目名 | 型 | 内容 |
|---|---|---|
| recognizedLatex | 文字列 | AIやOCRで認識したLaTeX |
| aiAnswer | 文字列 | AIが生成した答え |
| aiSteps | 配列 | AIが生成した途中式 |
| recognitionStatus | 文字列 | 未実行、成功、失敗など |

## 11. 保存方式

保存方式はIndexedDBを採用する。

| 理由 | 内容 |
|---|---|
| 画像保存に向く | PNGのData URLやBlobを保存しやすい |
| 容量面で有利 | localStorageより大きなデータを扱いやすい |
| オフライン対応 | ブラウザ内に保存できる |
| 無料 | 外部サービスを使わない |

## 12. 非機能要件

| 項目 | 要件 |
|---|---|
| 対応端末 | iPadを主対象とする |
| 対応ブラウザ | iPad Safariを主対象とする |
| 画面向き | 横向きを優先し、縦向きでも最低限使える |

| 保存 | IndexedDBに保存し、アプリを閉じても残る |
