# iPadネイティブ手書き数式ノートアプリ 要件定義書

## 1. 結論

Webアプリ版の仕様を、iPad単体で動くネイティブアプリ仕様に移植する。  
初期版ではAI連携やクラウド同期は行わず、手書き数式、LaTeX入力、数式表示、端末内保存を実現する。

## 2. 前提

| 項目 | 内容 |
|---|---|
| アプリ種別 | iPadネイティブアプリ |
| 主対象端末 | iPad |
| 入力方式 | 指、Apple Pencil |
| 初期版の通信 | 不要 |
| 初期版の課金 | なし |
| 保存方式 | 端末内保存 |
| デザイン | ノートアプリ風 |
| 画面向き | 横向き優先 |
| AI連携 | 将来対応。初期版では実装しない |

## 3. 目的

iPad上で数式を手書きし、学習記録として保存・確認できるネイティブアプリを作る。

Webアプリ版ではPCサーバーやネットワーク設定が必要になるため、iPad単体で安定して使えるネイティブアプリに移植する。

## 4. 移植理由

| 課題 | ネイティブ化による解決 |
|---|---|
| PCサーバーが必要 | iPad単体で起動できる |
| Wi-Fiやテザリング設定が面倒 | ネットワーク不要で使える |
| IPアドレスが変わる | URL共有が不要になる |
| Safariの制約がある | iPadOSの機能を直接使いやすい |
| Apple Pencil対応を強化したい | PencilKitを使える |

## 5. 推奨技術

### 5.1 第一候補

| 項目 | 内容 |
|---|---|
| 技術 | Swift / SwiftUI |
| 手書き入力 | PencilKit |
| 保存 | SwiftData または Core Data |
| 数式表示 | WKWebView + KaTeX、または画像/テキスト表示 |
| 配布 | Xcodeから実機インストール、TestFlight、App Store |

**推奨理由**

- iPadネイティブ機能と相性がよい
- Apple Pencil対応が強い
- PencilKitで手書き体験を作りやすい
- iPad単体アプリとして安定しやすい

### 5.2 代替候補

| 技術 | 向き | 注意点 |
|---|---|---|
| .NET MAUI | C# / Visual Studio利用者向け | iOS実機ビルドにはMac環境が必要になりやすい |
| React Native | JavaScript資産を活かしたい場合 | 手書きやPencilKit連携は追加対応が必要 |
| Flutter | クロスプラットフォーム開発 | iPad特化機能は個別実装が必要 |

## 6. 初期版の対象範囲

### 6.1 初期版で作る機能

| 機能 | 内容 | 優先度 |
|---|---|---|
| 手書きキャンバス | 指またはApple Pencilで数式を書ける | 必須 |
| ペン入力 | Apple Pencilで自然に書ける | 必須 |
| 消しゴム | 手書き線を消せる | 必須 |
| 戻る | 直前の操作を取り消せる | 必須 |
| クリア | キャンバス全体を消去できる | 必須 |
| タイトル入力 | ノート名や問題名を入力できる | 必須 |
| 数式入力 | LaTeXまたは通常テキストを入力できる | 必須 |
| 数式表示 | 入力したLaTeXを表示できる | 必須 |
| 答え入力 | 答えや解説を手入力できる | 必須 |
| 端末内保存 | ノート内容をiPad内に保存する | 必須 |
| 履歴一覧 | 保存済みノートを一覧表示する | 必須 |
| 詳細表示 | 保存済みノートを確認できる | 必須 |
| 編集 | 保存済みノートを編集できる | 必須 |
| 削除 | 保存済みノートを削除できる | 必須 |

### 6.2 初期版では作らない機能

| 機能 | 理由 |
|---|---|
| GPTによる自動解答 | API料金と通信が必要なため |
| 手書き数式の自動認識 | 初期版では実装負荷が高いため |
| クラウド同期 | 初期版ではiPad単体利用を優先するため |
| ログイン | 個人利用では不要なため |
| App Store公開 | 最初は実機確認を優先するため |

## 7. 画面要件

### 7.1 ノート一覧画面

保存済みの数式ノートを一覧表示する。

| 項目 | 内容 |
|---|---|
| 新規作成ボタン | 新しいノートを作成する |
| 検索欄 | タイトルで検索する |
| ノート一覧 | タイトル、更新日時を表示する |
| 削除 | 不要なノートを削除する |

### 7.2 ノート編集画面

手書き入力、数式入力、答え入力を行うメイン画面。

| 項目 | 内容 |
|---|---|
| タイトル入力 | ノートのタイトルを入力する |
| 手書きエリア | 数式や途中式を手書きする |
| ツールバー | ペン、消しゴム、戻る、クリア、保存を配置する |
| 数式入力欄 | LaTeXまたは通常テキストを入力する |
| 数式プレビュー | 入力した数式を表示する |
| 答え入力欄 | 答えや解説を入力する |

### 7.3 ノート詳細画面

保存済みノートを確認する画面。

| 項目 | 内容 |
|---|---|
| 手書き内容 | 保存した手書きデータを表示 |
| 数式表示 | LaTeX入力内容を表示 |
| 答え・解説 | 保存済みの答えや解説を表示 |
| 編集 | 編集画面へ移動 |
| 削除 | 確認後に削除 |

## 8. 機能要件

### 8.1 手書き入力

| 要件ID | 要件 |
|---|---|
| F-001 | ユーザーはApple Pencilまたは指で数式を書ける |
| F-002 | Apple Pencil入力時に自然な筆記感を提供する |
| F-003 | ペンの太さを変更できる |
| F-004 | ペン色は黒を初期値とする |
| F-005 | 消しゴムで一部を消せる |
| F-006 | 直前の操作を取り消せる |
| F-007 | キャンバス全体をクリアできる |
| F-008 | クリア時は確認メッセージを表示する |

### 8.2 保存

| 要件ID | 要件 |
|---|---|
| F-101 | 手書きデータを端末内に保存できる |
| F-102 | タイトルを保存できる |
| F-103 | 数式テキストを保存できる |
| F-104 | 答え・解説を保存できる |
| F-105 | 作成日時と更新日時を保存できる |
| F-106 | アプリを閉じても保存内容が残る |

### 8.3 履歴

| 要件ID | 要件 |
|---|---|
| F-201 | 保存済みノートを一覧表示できる |
| F-202 | 一覧にはタイトルと更新日時を表示する |
| F-203 | 保存済みノートを開いて詳細確認できる |
| F-204 | 保存済みノートを編集できる |
| F-205 | 保存済みノートを削除できる |
| F-206 | 削除時は確認メッセージを表示する |
| F-207 | タイトル検索ができる |

### 8.4 数式表示

| 要件ID | 要件 |
|---|---|
| F-301 | ユーザーはLaTeX形式の数式を入力できる |
| F-302 | 入力したLaTeXをプレビュー表示できる |
| F-303 | LaTeXエラー時はエラー表示する |
| F-304 | 通常テキストも保存できる |

## 9. データ要件

### 9.1 ノートデータ

| 項目名 | 型 | 必須 | 内容 |
|---|---|---:|---|
| id | UUID | 必須 | ノートID |
| title | String | 必須 | タイトル |
| drawingData | Data | 必須 | PencilKitなどの手書きデータ |
| drawingPreviewImage | Data | 任意 | 一覧・詳細表示用の画像 |
| formulaText | String | 任意 | LaTeXまたは通常テキスト |
| answerText | String | 任意 | 答え・解説 |
| createdAt | Date | 必須 | 作成日時 |
| updatedAt | Date | 必須 | 更新日時 |

### 9.2 将来拡張用データ

| 項目名 | 型 | 内容 |
|---|---|---|
| recognizedLatex | String | AIやOCRで認識したLaTeX |
| aiAnswer | String | AIが生成した答え |
| aiSteps | Stringまたは配列 | AIが生成した途中式 |
| recognitionStatus | String | 未実行、成功、失敗など |

## 19. 確信度

| 項目 | 確信度 | 理由 |
|---|---|---|
| iPad単体アプリ化で接続問題を避けられる | 高 | PCサーバーやWi-Fi接続が不要になるため |
| PencilKitが手書き用途に向く | 高 | Apple Pencil向けの標準的な実装候補 |
| 初期版を無料運用できる | 高 | 外部APIを使わなければ利用料は不要 |
| LaTeX表示をネイティブで完全再現する | 中 | WKWebView + KaTeXなど実装方式の選定が必要 |
| WindowsだけでiPadネイティブアプリを完結開発できる | 低 | iOS/iPadOS実機ビルドにはMacやApple開発環境が必要になる可能性が高い |
